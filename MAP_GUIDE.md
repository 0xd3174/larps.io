# Руководство по редактированию карт (Map Guide)

В этом проекте для создания и редактирования карт используется программа [Tiled Map Editor](https://www.mapeditor.org/). Карта сохраняется в формате JSON в файл `shared/map.json`.

## 1. Основные правила и структура
- Карта использует **Tile Layers** (Слои тайлов) для отрисовки графики и препятствий.
- Карта использует **Object Groups** (Слои объектов) для невидимых механик, таких как спавны игроков и порталы.
- Порядок слоёв в Tiled имеет значение! Слои отрисовываются строго снизу вверх (как они расположены в списке слоёв Tiled).

---

## 2. Тайловые слои (Tile Layers)
Названия слоёв напрямую влияют на логику игры. Вы можете называть слои как угодно, главное — использовать правильные суффиксы и подстроки.

### 2.1. Препятствия (`_solid`)
Если название тайлового слоя оканчивается на `_solid`, сервер будет считать **все нарисованные на нём тайлы непроходимыми препятствиями**. 
- Примеры: `walls_solid`, `water_solid`, `props_solid`.
- Если суффикса нет, игроки смогут свободно ходить по этим тайлам (например, слой `floor` или `grass`).

### 2.2. Отрисовка поверх персонажей (`_front`)
По умолчанию все тайловые слои отрисовываются **под** игроками. 
Если вы хотите, чтобы игрок мог визуально "зайти" за объект (например, спрятаться за кроной дерева или за крышей), добавьте в название слоя `_front`.
- Примеры: `bush_front`, `tree_front`, `clouds_front`.
- Такие слои будут отрисованы **поверх** всех персонажей.

### 2.3. Комбинации (`_front` + `_solid`)
Вы можете комбинировать эти два правила.
Например, слой `roof_front_solid` будет:
- Отрисован **поверх** игрока (благодаря `_front`).
- Непроходим для игрока (благодаря `_solid`).

---

## 3. Слои объектов (Object Groups)
Эти слои содержат не тайлы, а логические прямоугольники или точки. В Tiled они обычно делаются невидимыми (`visible: false`), так как нужны только серверу для логики.

### 3.1. Спавны (`spawns`)
Создайте слой объектов с названием `spawns`. 
- Внутри него создавайте прямоугольные объекты.
- Название объекта (Name) должно строго соответствовать роли: `Hider` (прячущийся) или `Seeker` (ищущий).
- Сервер случайным образом выберет одну из этих точек при старте игры для каждого игрока соответствующей роли.

### 3.2. Телепорты (`teleports`)
Создайте слой объектов с названием `teleports`.
- Создайте прямоугольный объект (зону телепортации).
- Зайдите в **Custom Properties** (Пользовательские свойства) этого объекта и добавьте два свойства типа `float`:
  - `targetX` — координата X, куда перенесётся игрок.
  - `targetY` — координата Y, куда перенесётся игрок.

---

## 4. Тайлест (Tileset)
Для отрисовки используется общий файл спрайтов.
- Файл изображения должен лежать по пути `frontend/public/graphics/tilesheet_complete.png`.
- При добавлении тайлсета в Tiled убедитесь, что путь относителен и указывает на этот файл, а параметры сетки (размер тайла) совпадают с настройками игры (сейчас `64x64`).
