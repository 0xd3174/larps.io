# Seekers vs Hiders

Многопользовательская веб-игра в прятки ("Прятки", "Prop Hunt"). Проект состоит из бэкенда на Go и фронтенда на TypeScript (Vite). Игроки делятся на прячущихся (Hiders) и ищущих (Seekers).

## Структура проекта

```text
├── backend/            # Исходный код сервера на Go
│   ├── internal/       # Основная логика: API, WebSocket, физика и карта
│   └── main.go         # Точка входа сервера
├── frontend/           # Клиентская часть на TypeScript + Vite
│   ├── public/         # Статика (графика)
│   └── src/            # Исходный код движка игры (рендер, сеть, ввод)
├── shared/             # Общие файлы для бэкенда и фронтенда
│   ├── config.json     # Общие настройки игры (радиус игроков и т.д.)
│   └── map.json        # Карта игры, созданная в Tiled Map Editor
├── MAP_GUIDE.md        # Руководство по редактированию карт
├── Dockerfile          # Инструкции для сборки полного контейнера с игрой
└── docker-compose.yml  # Файл для быстрого запуска через Docker Compose
```

## Как запустить проект (Запуск / Deploy)

Есть два основных способа запустить проект: с помощью Docker (рекомендуется для продакшена и быстрого старта) или локально в режиме разработчика (для внесения изменений).

### Вариант 1: Запуск через Docker (Самый простой)
Вам потребуется установленный Docker и Docker Compose.

1. Откройте терминал в корне проекта.
2. Выполните команду сборки и запуска:
   ```bash
   docker-compose up --build -d
   ```
3. Откройте в браузере `http://localhost:8080`.
4. Сервер будет отдавать и статику (фронтенд), и обрабатывать WebSocket соединения на этом порту.

Для остановки используйте команду:
```bash
docker-compose down
```

### Вариант 2: Локальный запуск (Для разработки)
Вам потребуются установленные **Go** и пакетный менеджер **npm** (или **bun** / **yarn**).

#### Шаг 1: Запуск бэкенда
Откройте первый терминал:
```bash
cd backend
go run main.go
```
Бэкенд запустится на порту `8080` (по умолчанию).

#### Шаг 2: Запуск фронтенда
Откройте второй терминал:
```bash
cd frontend
npm install   # или bun install / yarn
npm run dev   # или bun run dev / yarn dev
```
Vite запустит локальный dev-сервер (обычно на порту `5173`). 
В настройках `vite.config.js` уже прописан прокси, поэтому все запросы к `/api` и `/ws` будут автоматически перенаправляться на ваш запущенный Go-сервер (`localhost:8080`).

Откройте выданный Vite адрес (например, `http://localhost:5173`) в браузере.

## Редактирование карты
Чтобы изменить карту, редактируйте файл `shared/map.json` с помощью программы **Tiled Map Editor**.
Для ознакомления со слоями, правилами коллизии и логики спавнов — прочтите [MAP_GUIDE.md](./MAP_GUIDE.md).
